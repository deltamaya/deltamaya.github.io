elf文件格式

发表于 2023-07-25 分类于 CSAPP 阅读次数： 本文字数： 7.3k 阅读时长 ≈ 7 分钟
介绍elf文件的基本格式和信息

分区

elf文件有如下三个部分：elf首部、不同的sections、一个描述section信息的表

elf是Executable and Linkable format的缩写

首部

main.c:

test.c:

使用 gcc main.c -c指令对main进行编译和汇编，得到可重定位目标文件main.o

使用 readelf -h main.o查看main.o的elf首部信息：

其中第一行是elf的魔数，前四个字节表示这是一个elf文件，通俗来讲，魔数就是用来确定文件格式的

后面02代表运行环境的位数，01代表32位，02代表64位

第六个字节01代表字节序，01代表小端字节序，02代表大段字节序

第七个字节代表elf版本，一般是01

后面的七个字节没有定义，一般是0

我们可以看到Size of this header: 64 (bytes)表示这个elf首部的大小是64个字节

Type: REL (Relocatable file)表示这是一个可重定位目标文件，也就是.o或者.obj文件

section table

使用 readelf main.o -S查看section table的详细信息；

.text

这里用来存放编译好的机器代码，可以使用objdump来对这段代码进行反汇编查看

.data

这里用来存放一些已初始化的静态和全局变量

.bss

这里用来存放未初始化的静态和全局变量,以及初始化为0的静态和全局变量

注意，bss段只是一个占位符，实际上并不真正的占据空间

.rodata

这里用来存放一些read only data，一般是一些字符串字面量和switch语句的跳转表

使用 objdump -s test.o，可以查看到elf文件中具体的section的内容

可以看到data段有一个值为10，这就是我们定义的value, rodata 段有一个常量字符串，就是我们printf中的那个

符号表

我们可以使用 readelf -s main.o来查看main.o中的符号表，每一个.o文件都有一个符号表

我们可以看到在main中我们有x和y两个符号，这是我们定义的static变量，其中这个后缀是用来修饰变量名来避免命名冲突的。

我们对x进行了有效初始化，所以它处在data section，y处于bss section

其中的Ndx代表这个符号在section table的索引，value表示这个符号相对text section起始地址的偏移量（hex）

size表示符号所占字节数（dec）

最后是bind属性，local表示私有，只有当前文件可以进行全局访问，global表示全局，可以被其他文件引用

我们查看test.o的符号表

这里可以看到，test拥有全局的func符号，所以在链接过程中main中处于UNDefine的func符号找到了定义，产生可执行文件的时候会将这个符号进行合并，也就是符号表合并

同样的，test中声明了两个全局变量而不是静态变量，它们区别于x和y，它们是global的，因此main中也可以使用它们
