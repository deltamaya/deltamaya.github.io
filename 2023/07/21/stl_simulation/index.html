string

即使string并不是stl的一员，但是其使用并不比stl少，并且它的接口与stl相似，就简要介绍一下

成员可看作:

当string中存放的成员太少时它会放在暂存在buf中（栈上），ptr是在堆上开辟的空间

当向string中增加数据会增大size，直到size与capacity相等时会进行扩容，需要重新申请一个空间，然后拷贝字符串，再释放原来空间

npos是类中的一个静态常量（ps：类中有且只有静态整形常量才可以在类中直接定义）

接口：

contains

find家族

replace

c_str

to_stirng

getline

此外，string之间的比较是使用字典序进行比较的
当string进行复制的时候可能进行写时拷贝，也就是当发生拷贝的时候，只是先进行浅拷贝，当原string或者新string对内存进行写操作的时候，才会真正对string进行深拷贝，这样一定程度上减少了内存损耗

vector

vector中维护了三个指针，start对应起始地址，finish对应size处的地址

capacity对应最大容量的地址

vector的iterator行为与普通指针一致，无需额外封装，vector的iterator类型是RandomAccessIterator

接口：

reserve 预留空间，如果多就阔，少就不变

resize 输入目标size和填充值，少就pop元素，多就填充

shrink_to_fit 将capacity缩至size

data 返回数组起始位置的原始指针

可以看出list是一个带头双向循环链表
但是list的迭代器行为与普通指针不一致，所以我们就需要额外封装一层

list的iterator是双向迭代器，不支持std::sort，但是我们可以直接调用list的sort方法，虽然这个sort的速度还不如拷贝到vector去sort然后再拷贝回来快….

这里使用的Ref和Ptr是为了实现const_iterator，因为const_iterator的行为与iterator只有解引用的区别，所以我们使用类模板实现了这个效果，每个迭代器类都返回Ref但是模板参数中的Ref却不相同。
->操作符返回了解引用操作符返回值的地址，也就是返回了封装的指针，所以按理说应该写成 iter->->_member才合理，但是C++编译器对此进行了优化，使得可以简写为一个

void swap(List <T>&rhs){
std::swap(_header,rhs._header);
std::swap(_size,rhs._size);
}
List <T>& operator=(List <T> rhs){
swap(rhs);
return *this;
}

这是C++赋值的简要写法，并且回避了自我赋值的风险，将赋值的工作交给拷贝构造

deque

deque又叫双端队列

它同时支持头插头删尾插尾删，并且支持随机访问

它的底层大致如下：

deque有一个成员叫做中控数组，也就是一个数组指针数组，每个元素都指向一个在堆上开辟的数组

中控数组的使用是由中间向两边发散的，由于成员是中控数组，所以在拷贝或者扩容时的速度都非常快

但是它的下标访问就需要有一个算法来计算，所以随机访问的速度不如vector直接解引用

而且中间插入需要挪动数据，随机插入的速度不如list

但是它同时支持快速的头插头删和尾插尾删，所以用来给适配器stack和queue作为容器

接口：

基本与vector和list一致

priority_queue

优先级队列（可以理解为一个二叉堆）同样是一个适配器，每次堆顶的元素就是优先级最高的元素

所以，优先级队列不只需要一个容器，还需要一个比较函数

在c中，我们可以通过传递函数指针来实现这一点，但是C++提供了仿函数这个更加流畅的解决方案

这里通过使用传递进来的仿函数类型定义了一个对象，然后通过这个对象重载的 ()操作符，让他的行为像一个函数，也叫仿函数，我们通过这个仿函数来比较元素之间的优先级
